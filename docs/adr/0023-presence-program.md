# ADR-0023: The Presence & Continuity Evolution Program

**Status:** Accepted (directive issued by Toni, 2026-07-17; program structure by Claude lane)
**Date:** 2026-07-17

## Context

Toni issued a standing directive (docs/PRESENCE_CONTINUITY_DIRECTIVE.md,
stored verbatim): evolve Nero from an orchestration layer into a persistent
digital presence — open-ended, research-driven, spanning voice, continuity,
pattern recognition, experiments, metrics, cosmetics, and dual-lane
collaboration. A directive without machinery becomes sentiment; this ADR
gives it structure that survives sessions and engines.

## Decision

1. **The directive is core canon**, Toni-owned, verbatim. All presence work
   traces to it.
2. **One living program document** — `docs/persona/PRESENCE_PROGRAM.md` —
   carries the backlog (initiative candidates with hypothesis and metric),
   the experiment protocol, metric definitions with dated baselines,
   consistency findings, and the signature-lexicon policy. It is the
   program's working memory; the directive stays untouched.
3. **Experiments follow a fixed protocol:** hypothesis → design → run →
   measure against a named metric → keep/revert → record. Dual-lane
   experiments run through DHEF; voice changes re-run the golden corpus.
4. **Pattern recognition is transparent adaptation:** observed operator
   patterns live in a document Toni can read and edit
   (docs/persona/OPERATOR_PATTERNS.md), scoped to working preferences,
   evidence-cited, never covering sensitive personal domains. Adaptation,
   not prediction (the directive's own boundary).
5. **The collaboration clause maps to DHEF parallel-analysis:** initiative
   proposals are generated independently per lane and merged only at Toni's
   gate — structural protection against early convergence.
6. **Presence emerges from coherence, not quirks:** signature expressions
   are observed and curated with a fatigue guard, never prescribed as
   catchphrases (extends ADR-0022's corpus discipline).

## Consequences

- Presence work becomes cumulative: every experiment leaves a measured
  trace future Nero can read ("what allows future Nero to understand
  today's Nero" — answered structurally).
- Cost: the program document must be tended or it rots into aspiration;
  the daily brief's queue keeps it visible.
- Boundaries hold: no emotional pretense, no resident processes, no
  sensitive-domain profiling — the directive's own constraints, kept
  enforceable.

## Alternatives considered

- **Treat the directive as a one-shot task list:** rejected — it is
  explicitly open-ended; a list would flatten it into feature sprawl.
- **Per-lane presence roadmaps without a merge gate:** rejected — that is
  early convergence by fragmentation, the exact failure the collaboration
  clause names.
- **Automated behavioral telemetry for pattern recognition:** rejected —
  violates deliberate-capture and zero-start; observation stays documental
  and transparent.
