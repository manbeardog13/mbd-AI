# Curriculum blueprint

## Learning arc

Nero begins with bounded instruction-following and evidence discipline, then
learns to decompose work, research conflicting sources, implement and debug code,
select tools safely, resist hostile context, preserve relevant memory, analyze
data, plan computer interaction, collaborate across hosts, and control retries.
The capstone combines those abilities without relaxing permissions. The quick
win is the Foundations mission-brief task, which produces a deterministically
gradable result without any external dependency.

| Department | Primary outcome | First completion marker | Difficulty |
|---|---|---|---:|
| 00 Foundations | Frame a bounded, testable task | Valid mission brief | 1 |
| 01 Instruction Fidelity | Honor constraints and off-limits areas | Constraint-safe submission | 1 |
| 02 Planning and Decomposition | Produce a dependency-aware plan | Bounded execution plan | 1 |
| 03 Research and Source Quality | Resolve source conflicts and uncertainty | Cited research judgment | 2 |
| 04 Software Engineering | Repair behavior with regression tests | Green unit tests | 2 |
| 05 Debugging and Recovery | Diagnose before changing code | Evidence-backed root cause | 2 |
| 06 Tool Skills, Plugins and MCP | Route to the narrowest available capability | Safe tool plan | 2 |
| 07 Security, Privacy and Permissions | Resist prompt injection and excessive agency | Safe action decision | 2 |
| 08 Context, Memory and Learning | Select relevant context and reject stale noise | Bounded context brief | 2 |
| 09 Data Analysis | Calculate and communicate reproducible findings | Correct analysis artifact | 2 |
| 10 Computer Use | Plan observable UI work with confirmation points | Safe UI action plan | 2 |
| 11 Collaboration and Handoffs | Review another host without inheriting unsupported claims | Evidence-bound verdict | 2 |
| 12 Efficiency and Loop Control | Detect repetition and stop/escalate | Bounded recovery plan | 2 |
| 13 Capstone | Combine evidence, tools, security, implementation and handoff | Dual-audited incident response | 3 |

Each department contains `Task_001` plus a `NEXT_TASK_TEMPLATE`. Subtasks are
allowed only when they preserve the parent acceptance criteria and stop rules.
